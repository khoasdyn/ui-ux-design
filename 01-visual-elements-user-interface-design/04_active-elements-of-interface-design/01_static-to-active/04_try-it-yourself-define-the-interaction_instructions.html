In the previous week you defined the audience and the look and feel for your app. This week we’ll start to apply that look and feel to some interface elements or design patterns that you can use in your final interface sketches.

Before you begin, you need to think about the main interaction the user will have with your interface. Each stage of the interaction will visually guide the user, telling the user what to do, or how the app is working. But before we dive in it’s useful to map out what the interaction is step-by-step. You’ve already started this process in your description of the main activity or function of the app, and we can continue to do this non-visually, using words.

For a heartbeat app for marathon runners, a step-by-step or expanded description of the interaction could be something like:

At the end of a run, the user opens their heartbeat app on their phone. A visual cue on the screen instructs the user to place their thumb lightly over your phone’s home button. When their thumb is in the proper position, the app will cue a 15-second countdown timer on the screen. At the end of the 15 seconds, the timer will disappear and replaced with the heart rate.

For an app that searches for late-night sugary snacks within walking distance of a college campus, an expanded description could be:

A user would open the app and type in their location on their college campus, for example, the library. They would then select the maximum distance (radius) for their search using a slider (0ft to 1 mile). Once they have entered the parameters, they would press a “Search” button. The app would return a list of vending machines and their approximate distance away.

A shorthand version of the above could be:

Input location > Select max radius > Click Search > View result

Grab a piece of paper and quickly jot down some notes for your own project idea. This isn’t required, but a very helpful step before you start the assignment at the end of the week.
